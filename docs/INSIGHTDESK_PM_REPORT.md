# InsightDesk 产品报告（PM 汇报稿 · ENMGT 5400）

**文档版本：** 1.0  
**课程：** Cornell ENMGT 5400 · Final Project（演示与报告用）  
**产品名称：** InsightDesk（本地多智能体市场情报与策略工作台）

---

## 1. 电梯陈述与交付物说明（Elevator Pitch）

**InsightDesk** 是一款面向 **课程演示与本地研究** 的 Web 工作台：在 **用户本机** 运行后端与前端，通过 **多智能体编排**（Super Agent 分流、Planner、任务执行与流式对话）连接 **多种 LLM 提供商** 与 **行情 / 基本面数据源**，支持 **投资研究、资讯追踪、网格 / 提示词驱动策略** 等工作流。  

**最终交付形态：** 可运行的本地应用（浏览器访问 `http://localhost:1420`）+ 配套配置说明；**评分侧重 AI 编排与智能体能力**（课程 FAQ），界面实现为承载与演示手段。

---

## 2. 动机与目标（Why）

### 2.1 业务与个人动机

- **教学场景：** 需要在有限时间内展示「AI + 金融」的可运行原型，而非仅概念幻灯片。  
- **隐私与可控：** 密钥与对话默认落在本机配置目录，适合课堂与合规讨论中的 **数据驻留** 议题。  
- **智力驱动：** 探索 **多智能体分工**（研究、新闻、策略）与 **人机协同规划（HITL）** 在投研场景中的可行边界。

### 2.2 项目边界（与课程 FAQ 对齐）

- **领域开放：** 以通用「市场情报 + 策略实验」为主轴，可扩展到课程允许的叙事。  
- **工作量：** 与课程期望约 **30–35 小时** 的深度相匹配（搭建、配置、演示脚本与报告）。  
- **原创性：** 可在既有开源架构上演示 **迁移到新叙事（InsightDesk）与新演示脚本**，课程允许在非原创基础上的领域化复现（见课程说明）。

## 3. 产品理念与设计原则

| 原则 | 说明 |
|------|------|
| **本地优先（Local-first）** | 默认不依赖特定商业 SaaS；可选环境变量接入自有认证或云端排行，课堂版通常留空。 |
| **智能体编排优先** | 用户价值来自 **Super Agent → Planner → TaskExecutor → 流式响应** 的链路，而非单一聊天窗口。 |
| **可配置模型与数据** | 支持多家 LLM 与多数据源适配（具体见 `docs/CONFIGURATION_GUIDE.md`）。 |
| **透明与可审计** | 会话与任务状态可追溯，便于课堂讨论「AI 建议 ≠ 投资建议」。 |

## 4. 既有工作与现状起点（State of the Art）

### 4.1 产业与学术脉络

- **多智能体系统（MAS）：** 任务分解、工具调用、Agent-to-Agent 协议（如 A2A）已成为企业自动化与研究热点。  
- **金融垂直 Agent：** 研报摘要、因子式问答、交易辅助等均以 **合规与风控** 为前置约束。  
- **本地 / 私有化部署：** 满足 API Key 与数据不出域的需求，常见于机构试点与个人研究者场景。

### 4.2 本项目技术起点

- 后端采用 **Python + FastAPI**，核心编排见 `docs/CORE_ARCHITECTURE.md`（Super Agent、Orchestrator、Planner、流式事件与对话存储等）。  
- 前端为 **Bun + React Router + Vite** 的现代化 SPA，作为 **操作台与可视化层**。  
- 本仓库在课程场景下以 **InsightDesk** 为产品名做品牌化与课堂构建（Python 包目录名可能仍为历史命名，以兼容 import，属工程折中）。

## 5. 产品功能详述（What the product does）

### 5.1 用户能感知的核心能力

1. **统一工作台**  
   - 首页、自选股、市场与智能体入口、设置等模块；支持多语言界面（中 / 英 / 日等前端 locale）。

2. **多智能体协作（AI 核心）**  
   - **中枢 Hub 智能体（InsightDeskAgent）：** 汇总路由与研究 / 策略等相关能力入口（具体行为以后端 Agent 卡片与编排为准）。  
   - **研究类 Agent：** 基本面 / 文档研究与报告型输出（Research 方向）。  
   - **资讯类 Agent：** 新闻与推送类任务（News 方向）。  
   - **策略类 Agent：** 网格策略、提示词驱动策略等 **虚拟或实盘配置入口**（需在设置中配置交易所与模型）。

3. **模型与提供商配置**  
   - 在设置中配置 LLM 提供商与 API Key（详见配置指南）；用于驱动上述 Agent 的推理与工具调用。

4. **市场数据与自选**  
   - 通过后端适配层接入多数据源（如公开行情类适配）；支持自选列表与标的检索等课堂演示路径。

5. **会话与任务**  
   - 对话流式输出、任务状态与历史可追溯（具体交互以后端实现的会话 / 任务模型为准）。

### 5.2 刻意弱化或非评分重点（按课程 FAQ）

- **UI 工程细节** 不作为核心评分点；只要能稳定演示 AI 链路即可。  
- **实盘交易** 涉及交易所密钥与资金风险，课堂演示建议以 **虚拟 / 沙盒叙事** 为主，并明确免责声明。

## 6. 系统架构（便于课堂讲解 AI 部分）

以下摘自架构文档要点，汇报时可口述简化：

- **Super Agent：** 先对用户输入分流——可直接作答或交给 Planner。  
- **Planner + HITL：** 计划中断时可请求用户补充信息（人机协同）。  
- **TaskExecutor + A2A：** 远程 / 本地 Agent 任务状态回流，经 Event → Router → Buffer → 持久化 → UI 流式展示。  
- **存储：** SQLite 等与对话 / 策略相关的持久化（路径随系统应用目录，详见配置文档）。

## 7. 搭建流程（Installation & Build）

### 7.1 环境前提

- **macOS / Linux：** `bash`、`bun`、`uv`（`start.sh` 可在 macOS 上尝试通过 Homebrew 安装缺失工具）。  
- **Windows：** PowerShell 执行 `.\start.ps1`。

### 7.2 启动全流程（课堂演示推荐路径）

1. 克隆或解压课程仓库至本地。  
2. 在项目根目录执行：  
   ```bash
   bash start.sh
   ```  
3. 脚本典型行为：`uv sync`、数据库初始化、前端 `bun install`、`bun run dev`、后端 `python -m valuecell.server.main`。  
4. 浏览器访问：**前端** `http://localhost:1420`，**API** `http://localhost:8000`。  

### 7.3 配置文件位置（重要）

- **运行时 `.env`：** 通常位于系统应用配置目录（macOS 示例：`~/Library/Application Support/ValueCell/.env`——目录名可能仍为历史命名）；首次可由模板复制。  
- **前端可选：** `frontend/.env`（参考 `frontend/.env.example`）；课堂本地演示可将云端 OAuth / SaaS 相关变量留空。

### 7.4 模型与密钥

- 至少配置一种 LLM 提供商与密钥，否则 Agent 推理不可用；详见 **`docs/CONFIGURATION_GUIDE.md`**。

## 8. 使用方法（User Guide）

### 8.1 首次使用建议顺序

1. 启动前后端，确认健康检查或首页加载正常。  
2. 进入 **设置 → 模型提供商**，配置至少一个可用模型。  
3. 返回首页 / 智能体入口，选择 **研究 / 新闻 / 策略** 中与演示主线一致的 Agent。  
4. 输入结构化任务（例如：给定标的与研究问题），观察 **流式回复与任务状态**。  
5. （可选）在策略模块演示 **配置流程**，课堂场景避免绑定真实资金账户。

### 8.2 演示话术要点（PM 视角）

- **价值句式：** 「同一工作台把 **数据—推理—行动建议** 串起来；核心是 **多智能体编排与可替换模型**。」  
- **风险句式：** 「输出仅供教育与流程演示，不构成投资建议。」
